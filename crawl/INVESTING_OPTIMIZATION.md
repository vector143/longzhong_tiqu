# Investing.com 监控脚本 - 优化说明

## 🎯 优化内容

### 1. 智能停止机制

**优化前**：
- 检查固定的3页
- 即使都是旧文章也要全部检查完

**优化后**：
- ✅ 连续遇到5篇旧文章自动停止
- ✅ 某一页全是旧文章立即停止翻页
- ✅ 大幅减少不必要的请求

### 2. 更清晰的输出

**优化前**：
```
第 1 页新增: 0 篇
第 2 页新增: 0 篇
第 3 页新增: 0 篇
```

**优化后**：
```
第 1 页: 新增 2 篇, 跳过 33 篇
ℹ️  连续遇到 5 篇旧文章，停止检查
✅ 频道 commodities 完成: 新增 2 篇文章
```

### 3. 监控模式优化

**优化前**：
```
✅ 本轮完成: 共新增 0 篇文章
   - commodities: 0 篇
   - economic-indicators: 0 篇
   - economy: 0 篇
```

**优化后**：
```
✅ 本轮完成: 共新增 0 篇文章
   所有频道均无新文章
📊 数据库总计: 150 篇文章

😴 等待 300 秒后进行下一轮检查...
   下次检查时间: 11:50:00
   (按 Ctrl+C 停止监控)
```

### 4. 性能提升

| 场景 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 无新文章时 | 检查3页×35篇=105篇 | 检查5-10篇后停止 | **90%↓** |
| 有少量新文章 | 检查3页全部 | 遇到旧文章区域停止 | **50%↓** |
| 首页都是新文章 | 检查3页 | 检查3页 | 无变化 |

## 📈 性能对比

### 测试场景：无新文章时的性能

**测试条件**：
- 3个频道（commodities, economic-indicators, economy）
- 已爬取100篇文章
- 所有文章都已存在

**优化前**：
```
总请求数: 3频道 × 3页 = 9次列表请求
总检查数: 9页 × 35篇 = 315篇文章
总耗时: 9 × 3秒 = 27秒
```

**优化后**：
```
总请求数: 3频道 × 1页 = 3次列表请求
总检查数: 3页 × 5篇 = 15篇文章（遇到5篇旧文章停止）
总耗时: 3 × 3秒 = 9秒
```

**性能提升**：
- 请求数减少：67% ↓
- 检查数减少：95% ↓
- 耗时减少：67% ↓

## 💡 使用建议

1. **首次运行**：先爬取历史文章（100-200篇）
   ```bash
   python crawl/investing_monitor.py --history 100 --proxy http://127.0.0.1:7897
   ```

2. **日常监控**：使用持续监控模式，间隔5-10分钟
   ```bash
   python crawl/investing_monitor.py --monitor --interval 300 --proxy http://127.0.0.1:7897
   ```

3. **手动更新**：需要立即检查新文章时
   ```bash
   python crawl/investing_monitor.py --proxy http://127.0.0.1:7897
   ```

4. **调整参数**：根据实际情况调整
   - `--interval`: 检查间隔（秒），建议300-600
   - `--delay`: 请求延迟（秒），建议2-3
   - `--channels`: 只监控需要的频道

## ⚠️ 注意事项

1. **代理必须**：必须使用 `--proxy` 参数，否则会被重定向到中文站点
2. **延迟设置**：`--delay` 不要小于2秒，避免被封IP
3. **监控间隔**：`--interval` 建议不小于300秒（5分钟）
4. **去重数据库**：不要删除 `investing_articles.db.json` 文件
